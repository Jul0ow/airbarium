# Lot 8e — Helm chart + CronJob + Dockerfile Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Package the Airbarium backend for Kubernetes — one Docker image serving both the API and the one-shot purge cron, plus a minimal Helm chart deploying the API Deployment, the purge CronJob, a migration hook Job, and an optional Gateway API HTTPRoute.

**Architecture:** Single multi-stage Bun image. Single Helm chart `deploy/helm/airbarium-api/` with the API Deployment + Service, a CronJob (`bun run cron`), a `pre-install`/`pre-upgrade` hook Job running migrations (`bun run db:migrate`) with its own hook-scoped ConfigMap/Secret copies, and a toggleable HTTPRoute. Postgres/Garage/Gateway are external prerequisites. Env split: non-sensitive → ConfigMap, sensitive → Secret (or `existingSecret`).

**Tech Stack:** Bun, Helm 3, Kubernetes (Deployment, CronJob, Job, Gateway API HTTPRoute), Docker. Verification via `hadolint`, `helm lint`, `helm template | kubeconform` (no Bun unit tests for YAML).

Reference spec: `docs/superpowers/specs/2026-06-14-lot-8e-helm-design.md`.
