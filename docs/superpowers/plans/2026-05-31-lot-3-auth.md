# Plan d'implémentation — Lot 3 Auth (Airbarium backend)

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land Better Auth (email + password), an SMTP mailer, the `/v1/auth/*` surface, `GET/PATCH /v1/me`, and an `authMiddleware` that gates everything else.

**Architecture:** Better Auth handles all `/v1/auth/*` requests (its own router mounted in front of our routes), uses the Drizzle adapter against three new tables (`account`, `session`, `verification`) plus the existing `users` table from Lot 2. A thin `nodemailer` wrapper delivers verify-email and reset-password mails (MailHog in dev, real SMTP in prod). Our own routes consume Better Auth's session via `authMiddleware` which populates `c.var.user` / `c.var.session`.

**Tech Stack:** Better Auth 1.x with Drizzle adapter + bearer plugin, nodemailer, Bun.password (Argon2id), Zod, Hono.

Lot 1 shipped the Hono skeleton; Lot 2 shipped Drizzle schemas + migrations + DB-probing health. Lot 3 brings auth online — every subsequent lot (specimens, identifications, RGPD delete) depends on it.

The spec at `docs/superpowers/specs/2026-05-31-airbarium-lot-3-auth-design.md` fixes the design decisions. Re-read it once before starting if you're unfamiliar.

## Décisions ré-affirmées (avant écriture du code)

- **`users` table NOT modified.** Better Auth is configured via `user.modelName = 'users'` and `additionalFields` for `avatarUrl` + `deletedAt`. No `image` column.
- **BA schemas are hand-written**, not generated by `@better-auth/cli`. Keeps snake_case discipline + avoids a dev dep we don't otherwise need.
- **Password hasher = `Bun.password.hash(pw, { algorithm: 'argon2id' })`** with `Bun.password.verify(password, hash)` for the verify side. Override BA's scrypt default.
- **Bearer plugin enabled from day 1.** Mobile flow consumes `body.session.token` from sign-in/sign-up response.
- **Rate limiter = Better Auth's built-in, `storage: 'memory'`** for now. Lot 8 swaps to DB-backed via `secondaryStorage`.
- **Email verification NOT required to sign in.** `sendOnSignUp = true` only.
- **Trusted origins = CORS origins** (keep in sync manually for now; CLAUDE.md note added in Task 19).
- **Cron worker entry point (`src/cron.ts`) stays out of Lot 3.** The `cron` script referenced in CLAUDE.md is a Lot 8 deliverable — that mismatch is pre-existing and unrelated to this lot.

```ts
import nodemailer, { type Transporter } from 'nodemailer';
import { env } from '@/config/env';

export type SendMailInput = {
  to: string;
  subject: string;
  html: string;
  text?: string;
};

type SendMailImpl = (input: SendMailInput) => Promise<void>;

let transporter: Transporter | null = null;

function getTransporter(): Transporter {
  if (!transporter) {
    transporter = nodemailer.createTransport(env.SMTP_URL);
  }
  return transporter;
}

const defaultImpl: SendMailImpl = async (input) => {
  try {
    const result = await getTransporter().sendMail({
      from: env.MAIL_FROM,
      to: input.to,
      subject: input.subject,
      html: input.html,
      text: input.text,
    });
    console.log(
      JSON.stringify({
        level: 'info',
        msg: 'mail.sent',
        to: input.to,
        subject: input.subject,
        messageId: result.messageId,
      }),
    );
  } catch (err) {
    console.error(
      JSON.stringify({
        level: 'error',
        msg: 'mail.failed',
        to: input.to,
        subject: input.subject,
        err: String(err),
      }),
    );
  }
};

let impl: SendMailImpl = defaultImpl;

export async function sendMail(input: SendMailInput): Promise<void> {
  return impl(input);
}

// Test-only swap. Returns a restore fn that resets to the default implementation.
export function __setSendMailForTests(replacement: SendMailImpl): () => void {
  const prev = impl;
  impl = replacement;
  return () => {
    impl = prev;
  };
}
```

The `__setSendMailForTests` setter is the only sanctioned way to override `sendMail` — Better Auth's config captures the exported `sendMail` reference at import time, so we route every call through the swappable `impl` variable. Logging stays on `console` here (no need to depend on the pino instance from Lot 1, keeps the mailer testable in isolation).

```ts
    const result = await auth.api.getSession({ headers: c.req.raw.headers });
    if (!result?.user || !result?.session) {
      throw new UnauthorizedError('Authentication required');
    }
    c.set('user', result.user);
    c.set('session', result.session);
    await next();
  };
};
```

```ts
  };
```

```ts
  name: string;

    id: u.id,
    email: u.email,
    email_verified: u.emailVerified,
    name: u.name,
    avatar_url: u.avatarUrl ?? null,
    created_at: u.createdAt.toISOString(),
  };

export async function getMe(userId: string): Promise<ProfileResponse> {
  const [row] = await db.select().from(users).where(eq(users.id, userId));
  if (!row) throw new Error(`profile.getMe: user ${userId} not found`);
  return toResponse(row);
}

export async function updateMe(
  userId: string,
  patch: { name?: string },
): Promise<ProfileResponse> {
  const [row] = await db
    .update(users)
    .set({ ...patch, updatedAt: new Date() })
    .where(eq(users.id, userId))
    .returning();
  if (!row) throw new Error(`profile.updateMe: user ${userId} not found`);
  return toResponse(row);
}
```
